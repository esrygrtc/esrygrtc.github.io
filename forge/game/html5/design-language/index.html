1. Product sentence

A bright metropolitan case file made tactile: powder-blue map paper, navy casework, cream evidence cards, warm gold earned rewards, and green action controls — softly dimensional, orderly, optimistic, one-thumb readable.

Fiction: noir-precinct — cops, thief, city grid, case files. Look: premium bright casual. The noir lives in names, scenes, and characters, never in the palette. Not police-noir grime. Not parchment fantasy. Not casino. Not generic candy UI.

3. Logo

One logo. No alternates, no per-screen redraws. Final render only after this language is ratified (non-goal until sign-off).

  1. Wordmark: COPDOKU, Nunito Sans 900 caps, navy-900, tracking +2%.
  2. The O is the badge: gold disc (gold-500 body, gold-700 rim, gold-300 top highlight) carrying a navy 5-point star — the gameplay officer glyph promoted to gold (earned/premium reading). Badge replaces the first O.
  3. Subtitle slot (optional): case name in caption, navy-700, caps, centered. Never part of the base lockup.
  4. Construction: badge diameter = cap height; star inscribed at 55% badge diameter; clear space = cap height of C on all sides; minimum digital width 120px.
  5. App icon / favicon: map-100 squircle (R=32 family), badge-O centered at 60% size; optional 1px map-300 map-line arc behind, ≤20% contrast.
  6. Placement: HOME hero top-center; result cards and HUD never repeat it.
  7. Don'ts: no noir/handcuff/fingerprint clipart, no gradient letters outside tokens, no script or condensed face, no outline-only variant, no rotation, no shadow beyond E1 on the badge.

4.2 REGION CUE — pattern + fill, never hue alone (binding law, #141 §5 · #130 §4)

Regions (fiction: precincts) are double-coded on two independent axes: (1) fill tint (hue family + stepped value) and (2) border pattern shape. Either axis alone must distinguish all regions; grayscale kills axis 1, so axis 2 always carries identity. This is an accessibility requirement, not a style opinion.

B dashed 10 6 #FBE7E7 #D95B5B light
C dotted 0.5 8 round cap #E6F5EA #45A866 mid-light
D double (two 1.5px strokes ±1.6px) #FBF2DC #D9A52E mid
E dash-dot 12 4 2 4 #F0EAFB #8B5CF6 mid-dark
F long-dash 18 7 #DFF3F5 #2AA5B3 darker

Render rule: border drawn 3px, inset 1.5px INSIDE the region's own cells; a shared edge = two parallel patterned strokes (one per region) — patterns never overlap. Board outer frame is a separate navy-900 stroke. Provenance: #141 R1 sketch sheet (ef51bebd), implemented and verified in the P1 greybox. 7. Materials, light, elevation

Physics per MASTER_ART_PROMPT §3.2/§3.5: one light, soft top-center; highlight above, local-hue shade below; no black shadows.

  • Material stack (one only): matte map paper → cream evidence card → navy structural frame → glossy toy-plastic interactive control → gold earned reward.
  • Elevation: E0 flat map field · E1 card 0 3px 0 rgba(12,47,80,.14), 0 8px 18px rgba(12,47,80,.12) · E2 button/popup 0 5px 0 rgba(12,47,80,.22), 0 12px 24px rgba(12,47,80,.18). Interactive surfaces add 1px top highlight rgba(255,255,255,.55).
  • Radius family: R=8 base — chip 8, button/input 16, card 24, hero/modal 32, circle 999. No arbitrary radii.
  • Spacing grid: 4/8/12/16/24/32/40 only. Side margin 16px at 390px; card padding 24 (16 compact). Rims 2px; focus 3px; dividers 1px.

8. Component anatomy

Every recipe obeys: tap targets ≥44×44px, layouts proven at 390×600 and 390×844 (P7 device truth).

8.1 Buttons — one family

Outer local-hue rim → vertical body gradient → top highlight → bottom core shade → centered rounded-bold label/icon → E2 shadow. Primary CTA height 56px, compact 48px. Variants: primary green (single proceed action) · secondary cream/navy · utility navy + white icon · reward gold (value only) · destructive red (danger only) · disabled map-gray 55%. States: default · pressed = translateY(3px), shadow reduced, gradient flattened · focus = 3px map-300 halo · disabled · loading.

8.2 Panels & modals

Navy frame + cream face + optional pale-map inset; modal = scrim + 32px hero card + integrated title + one bottom action row + standard 44px utility close. No parchment, royal-gold, glass, or flat-web cards.

8.3 HUD counters

One pill: 40px high, cream, navy text, left 32px icon token, 2px navy rim. Values never float. Coin pill is the destination of all coin-flight motion and stays visible during win/fail panels.

8.4 Toggles & settings rows

Toggle: 52×32 track (green-400→600 on / desat map-300 off), 24px cream knob E1, 2px rim. Row: 56px, cream, left icon token, body label, toggle right, 1px cream-200 divider. Toggles are the only on/off control.

8.5 Board cells & states

Board = highest-contrast central object. Region fills per §4.2. Manual X is #000000 (sole pure-black exception, standing gameplay rule). Every state change uses ≥2 channels (fill + border/icon), never color alone: empty · mark ✕ (player: 2.5–3px #5A6472, 56% cell) · auto-✕ (system: 2px #9AA4B0, 44%) · officer (§6 glyph, 60% cell) · thief reveal · thief ring (3px #F2B33C).

8.6 Stars, hearts, price tags

Stars §6, earned gold / unearned map-300. Hearts §6. Price tag = gold pill + counter text; real-money price always visible before any confirm (P9).

9. Motion principles

Grammar here; exact numbers are PULSE's envelope (feel.json, game-feel §3 — transcribe, never improvise; an improvised easing is a defect).

  1. One easing family per surface — board = easeOutCubic throughout; mixed families read cheap.
  2. Ack first: every tap acknowledges ≤50ms, from pointerdown, before rules work. UI press: 80–100ms down, 120–160ms return.
  3. Panels: 180–240ms .96→1 + fade in; 140–180ms out.
  4. Reward: 700–1200ms deterministic coin flight, gold only; HUD pulse 1.0→1.15→1.0.
  5. Error: red state + 180ms shake, no sound above tier 1. Fail: 200–300ms soft desaturate + settle — dignity, not punishment. Win: 300–600ms bloom; input never locked past 600ms.
  6. Material harmony (LEARNING 2): motion must deliver the weight the render promises — toy-plastic pops small and fast, paper slides, gold flies with mass. Art that promises weight motion can't deliver is dissonance = cheap.
  7. Reduced motion: settles ≤100ms, final state and balance preserved.

  • THIEF / REVEAL (two-stage, owner msg 18995): same map/card/button system; the board MORPHS shape keeping the same precinct colors/patterns; thieves ≈ half cop count, double-tap verb; X-marked blocks persist.   • LEVEL COMPLETE: navy frame + cream card + map inset (no separate celebration style).   • LEVEL FAIL: scrim + hero card; board settles 200–300ms. CASE COLD + one body line ("The trail went cold — same case, fresh start.") · 3 empty heart pips · top-left utility close → HOME (owner's exact route) · one green RETRY · no penalty copy, no revive-for-coins, no ads.

  • CLUE / ECONOMY MODALS: standard anatomy; inventory + consequence + price in body/counter; gold = coin value; green confirms; cream cancels.

11.1 Mandatory language header (paste verbatim after §1 CORE, before negatives)

COPDOKU CITY CASE v3 — binding visual language (docs/art/COPDOKU_VISUAL_LANGUAGE.md). Powder-blue matte map field, navy structural frames, cream evidence-card faces, warm gold exclusively for earned rewards/value, green exclusively for primary proceed/confirm actions, red exclusively for error/destructive. Region/precinct cue = pattern+fill double-code, never hue alone. One top-center light, one toy-plastic interaction material, one matte paper/card material, one R=8 radius family (8/16/24/32), one Nunito Sans extra-bold type family, one rounded-filled icon family, one button anatomy. Logo: COPDOKU caps with gold badge-O (navy star). 4/8px spacing, 16px phone margin, controls ≥44px. The board is the highest-contrast object. Composition may change; tokens and component anatomy may not.

11.2 Mandatory negative block (append to §2 NEGATIVE)

NOT: screen-specific visual language, bespoke button silhouette, mixed corner radii, mixed fonts, thin-line icons mixed with rendered icons, floating text or counters, unsupported colors, black drop shadows, parchment-fantasy popup, royal-gold frame, glassmorphism, flat web-dashboard card, police-noir grime, casino reward styling, countdown offers, ribbons, badges, gradients not in CITY CASE v3, hue-only region coding. Do not imitate a reference app's trade dress.
